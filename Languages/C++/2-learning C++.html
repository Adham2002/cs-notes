First Example

public class Money {

euros = e; cents = c;

}

public int getEuros() {

return euros;

}

public int getCents() {

return cents;

Equivalent class in C++

  • Class definition is usually created in two files.
  • In the lectures we have been using a single file to keep the examples simple, but (like C) we should separate out the declaration and the implementation.
  • The header (.h) file specifies structure of class
  • Implementation file (.cpp) provides code for methods. In this file we define member function's bodies with this notation:

void ClassName::methodName()

{

        // code goes here:

}

  • Accessibility of class members is specified via public and private section headings in header file
  • Methods that dont change an object should be declared as const, in both files

Initialiser lists and inlining to create shorter code

  • Initialiser Lists
  • Often a constructor will just set member variables.
  • In our example the constructor Money::Money does this:

Money::Money(int e, int c)

{

euros = e;

cents = c;

}

  • We can change it to use an initializer list to do this automatically

Money::Money(int e, int c): euros(e), cents(c) {}

  • Now our constructor fits on a single line

  • Inlining
  •  Simple methods can be implemented directly in the header (.h) file, as part of the class definition

// Money.h:

class Money

{

        // these inlined methods can be removed from Money.cpp

public: Money(int e, int c): euros(e), cents(c) {}

int getEuros() const { return euros; }

int getCents() const { return cents; }

std::string toString() const;

// this is a longer function and should still be in Money.cpp:

Money plus(const Money&) const;

private: int euros, cents;

};

  • This is an optimisation hint to the compiler - suggesting that it can substitute the code of the method body in places where the method is called
  • A potentially useful way of avoiding the normal overhead associated with calling a method or function
  • Compiler is free to ignore the suggestion!
  • The general "rule of thumb" is that if the method's code is more than 2 lines long (or has complex operations) then it should not be in the header file

Operator Overloading

Unlike Java, in C++ we can change the definition of operators (such as +, -, << etc) for our classes

  • the class of the object on the left of the operator determines which class is queried for the operator.
  • almost all operators can be overloaded (some exceptions include sizeof, typeid, member function dot ., and the conditional operator ? :)
  • polymorphism allows us to create different behaviour for different parameter types
  • of course, this must be used carefully to not make your code more difficult to understand: more power; more responsibility!

Throwing Exceptions in C++

like Java, C++ has exceptions.

  • You can throw and catch exceptions in C++ like Java
  • You don't have to declare that you throw Exceptions (unlike Java Exceptions but like Java Errors).
  • You can throw any type (including primitives such as int and double)
  • There are many built-in examples such as std::bad_alloc and std::bad_cast.
  • To continue our earlier example we could write:

#include<stdexcept>

Money::Money(int e, int c)

{

if (e < 0) {

throw invalid_argument("invalid euros"); // returns from the method instantly

}

if (c < 0 || c > 99) {

throw invalid_argument("invalid cents");

}

euros = e; cents = c;

}

now if we call

Money m1 (23, 101);

we get an an exception in the output:

terminate called after throwing an instance of 'std::invalid_argument'

  what():  invalid cents

  • To catch the error use try over the statement that throws an error
  • And a catch block to say what to do when the exception is thrown
  • The what() method on an exception often gives a text description of what went wrong.
  • For example:

try {

Money m1 (23, 101);

} catch (invalid_argument const& e ) {

        cout << "invalid argument " << e.what() << endl;

}

Inheritance in C++

  • Recall that inheritance means that a subclass has the same contract (methods, fields etc) as the superclass, but
  • might do some things differently, and
  • might declare additional methods and fields.
  • C++ supports inheritance between classes (just like Java).
  • This allows subclasses to change the behaviour of the superclasses.
  • For example: a subclass may have a different method body.
  • C++ supports multiple inheritance: for example a subclass can inherit from 2 superclasses and access all protected member variables in both classes.
  • Multiple inheritance can cause subtle bugs and issues: avoid it if you can.
  • To declare a subclass (Foo) class which extends the superclass (Bar) we write:

class Bar {

};

class Foo : public Bar {

};

  • the public here says that we can cast from a Foo to a Bar. For example, a method void flibble(Bar b1){} can be called with a Foo or a Bar object.
  • in UML we would describe the relationship between Foo and Bar like this:

  • If we want the subclass to override a method in the superclass, then we must declare the method virtual.

class Bar {

    virtual string getName() {

        return "I am Bar";

    }

};

class Foo : public Bar {

    string getName() {

        return "I am Foo";

    }

};

  • Like Java, C++ has abstract classes. Sometimes we don't want the superclass to provide a method body, only the signature. In the superclass we can add = 0 to do this:

class Bar {

    virtual string getName()=0;

};

class Foo : public Bar {

    string getName() {

        return "I am Foo";

    }

};

  • As Java, if a C++ superclass has one or more constructor, the subclass must call one of them. Here Foo has two constructors, which both call the superclass constructor.

class Bar {

public:

    string name;

    Bar(string n) : name(n){}

};

class Foo : public Bar {

    Foo(string n) : Bar (n) {}

    Foo() : Bar ("FUBAR") {}

};

Templates in C++

  • C++ allows the use of templates, they fulfil a similar purpose to Java's generics.
  • They allow to write more general code
  • Classes with can have varying base types
  • For example, we can have a matrix of type T:

class matrix <T> // T is the templated type

{

int nRows, nCols;

T **coefficients;

};

  • and create instances with:

matrix<float> floatMatrix;

matrix<double> doubleMatrix;

  • C++'s std::containers may be used with templates:

std::vector<int> v = {7, 5, 16, 8};

std::map<std::string, int>;

  • Note that we can use primitive types (int, double etc..) as template types - something Java can't do.

Object lifecycle and copying

  • In C++, like C, we have to be aware of memory management for our classes.
  • The destructor is the opposite of a constructor: it is called to free the resources that an object has claimed dynamically:

Currency(int, int) { /* do dynamic allocation here with new */ }

~Currency() { /* free allocated values here with delete */ }

  • One example is a class which wraps a variable length array.
  • Because the array is a variable length, they cannot be member variables in a class (C++ classes are just like C's structs in this way).
  • Therefore we must allocate dynamic space for them using the new keyword
  • When we are done with them, we must also use array variation of the delete[] keyword with square brackets.
  • Here is a new version of the Currency class (CurrencyArray) which stores the euros and cents in an array. We will allocate it dynamically, imagining that we may need to extend it "pounds, shillings, and pence" (needs an array of length 3), or Japanese Yen (needs an array of length 1).

class CurrencyArray

{

public:

        int *amountArray;

        CurrencyArray(int e, int c) // Constructor

{

                amountArray= new int[2];

                amountArray[0] = e;

                amountArray[1] = c;

        }

        int getEuros() const

{

                return amountArray[0];

        }

        int getCents() const

{

                return amountArray[1];

        }

        ~CurrencyArray() // Destructor

{

                delete[] amountArray;

        }

};
